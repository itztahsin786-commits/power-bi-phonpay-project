📊 PhonePe Payment Analytics Dashboard
🧾 Overview

This Power BI project visualizes and analyzes digital transaction data from PhonePe, a popular digital payment platform.
The dashboard provides key insights into payment trends, transaction performance, and failure reasons across different services such as Loans, Insurance, Money Transfers, and Recharges/Bills.

🎯 Objective

The main goal of this dashboard is to:

Track total and successful transaction amounts.

Identify and analyze failed payment reasons.

Compare service-wise and month-wise transaction patterns.

Provide actionable insights to improve digital payment reliability and performance.
---------------------------------------------------------------------------------------------
📊 Dashboard Features
1. KPI Cards

Total Amount: ₹2,894M

Successful Amount: ₹2,774M

Failed Amount: ₹120M

Total Transactions: 249.79K

2. Date Range Filter

An interactive date slicer (from 3/2/2024 to 12/30/2024) allows users to filter transactions over time.

3. Failure Analysis

Pie Chart: Displays the percentage of failed payment reasons:

Server Error

Wrong Info

Bank Denied

Insufficient Balance

Wrong PIN

Bar Chart: Service-wise failure amount (Loans, Insurance, Money Transfer, Recharge & Bills).

Line Chart: Month-wise failure trend across the year.
------------------------------------------------------------------------------------------------------
📈 Insights

The Loans service has the highest failure amount among all services.

Server errors and bank denial are the top reasons for payment failures.

Transaction failures peak around July and September, indicating possible seasonal or server load issues.

Despite failures, overall success rate remains high, with ₹2,774M successfully processed out of ₹2,894M.

🧠 Tools & Technologies

Power BI Desktop

Data Modeling (Relationships, DAX, Measures)

Visualizations (KPI Cards, Pie Chart, Line Chart, Bar Chart)
